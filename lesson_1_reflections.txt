你认为，手动选择何时创建提交（像在 Git 中做的那样）与自动保存版本（如 Google Docs 所做的那样）各有何优缺点？
手动提交：优点是可以根据自己的需求来进行保存，并且可以进行适当注释
	  缺点是可能出现意外状况
自动提交：优点是自动为你保存
	  缺点是可能保存不必要的文件，甚至可能是错误的。

何你认为一些版本控制系统（例如 Git）允许在一次提交中保存多个文件，而其他系统（例如 Google Docs）单独对待每个文件？
因为GIT 面向的是程序员，而一个程序就是对各文件共同运作而进行的，所以多个文件的跟踪会使改错，查看自己的进程变得更加容易。
而Word等其他系统面对的时日常生活，我们常常用一个文件来包含所有内容，因此多文件跟踪时没有必要的。

如何使用命令 git log 和 git diff 来查看文件的历史记录？
git log是用来查看提交历史的，加上 --stat可以详细查看修改的详细信息
